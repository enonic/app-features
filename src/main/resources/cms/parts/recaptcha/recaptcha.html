reCAPTCHA Lib Demo

This part exercises lib-recaptcha. The site key and secret key are read from the site config (fields recaptchaSiteKey and recaptchaSecretKey). Token verification is delegated to recaptcha.verify(), which calls Google's siteverify endpoint.

Active variant: . Threshold (v3): .

  • Append ?variant=v2 or ?variant=v3 to switch widget.
  • Append ?threshold=0.7 to override the v3 score threshold.
  • Append ?simulatedScore=0.9 to test threshold logic without a real v3 token.
  • Append ?forceFail=true to verify a deliberately invalid token (server-side only, no widget).
  • Append ?format=json for a machine-readable result.

Note: the form below only works when the page is opened in Preview (or live) mode. When the part is rendered inline — such as in Content Studio's edit view — the submit POSTs to the isolated component URL and does not round-trip, so no verification result is produced. Open the preview to submit a token.

Submit a token

reCAPTCHA v3 runs invisibly. A token is generated on submit via grecaptcha.execute() and posted as g-recaptcha-response.

Configure recaptchaSiteKey and recaptchaSecretKey in the site config to render the widget and verify tokens. The form is shown here so the demo remains visible.
